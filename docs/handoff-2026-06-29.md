## Project Context

**wtcc — WorkTree Command Center**: a Rust 2024 TUI (`ratatui` + `crossterm`) that orchestrates
Claude Code agents across git worktrees — a Linux reimagining of the macOS app
[Supacode](https://github.com/supabitapp/supacode). It runs *inside* the host terminal (built/tested
on ghostty on Arch/Wayland); it does **not** embed libghostty (not embeddable in a Linux GUI today).

### Architecture (TEA-style: model → update → view)
- Domain (pure, argv-only subprocess): `config.rs` (XDG TOML), `repository.rs`, `worktree.rs`
  (porcelain parse + `add_new_branch`/`add_existing_branch`/`branch_exists`/`remove`/`slugify`),
  `vcs.rs` (git dirty + `gh` PR/CI badges, off-thread via mpsc), `session.rs` (per-worktree PTY in
  `tmux new-session -A`, `vt100` parser, activity tracking, `kill`/restart).
- UI: `ui/mod.rs` (layout + overlays: palette/input/confirm/help), `ui/sidebar.rs`
  (`sidebar_rows()` = single source of row ordering, shared with mouse hit-test), `ui/statusbar.rs`,
  `ui/palette.rs`.
- Shell: `app.rs` (App state, actions), `event.rs` (`handle_key`/`handle_mouse`/`hit_test`, HELP
  const tables), `main.rs` (CLI flags via `parse_args`, terminal lifecycle, panic-safe teardown,
  draw/poll/update loop with mouse capture).

## Key Decisions & Gotchas

- **Terminal stack is a coordinated set** — `ratatui 0.30` / `crossterm 0.29` / `tui-term 0.3` /
  `vt100 0.16` move together (unicode-width conflict otherwise). See ADR `docs/adr/0001-*.md`.
- **argv-only subprocess** is the project's core security invariant: never `sh -c` with interpolated
  repo paths / branch names. Branch names are untrusted; slugify only the derived filesystem path,
  never the literal `-b`/branch argument. CLAUDE.md enforces this; CodeQL + reviews check it.
- **AUR auto-publish:** pushing a `vX.Y.Z` tag triggers `release.yml` → GitHub Release + AUR. The AUR
  push hook on the dev machine blocks `push ... master`; the GitHub Action does the AUR push using
  the `AUR_SSH_PRIVATE_KEY` repo secret. cgit/RPC index lags — verify AUR via `git clone https://aur.archlinux.org/wtcc.git`.
- **Local commit rule:** the user's global CLAUDE.md forbids auto-commit; the auto-mode classifier
  blocks commits without explicit confirmation. Get an OK (or a batch mode) before committing.
- **Verify with a forced rebuild:** `cargo clean -p wtcc && cargo clippy --all-targets -- -D warnings`
  — cached clippy hid a real CI failure once.
- **Functional checks via tmux:** `scripts/smoke-test.sh` and `docs/demo/capture.sh` drive the release
  binary headlessly (register repo → agent → render) — reuse this pattern to validate UI behavior.
- **/issue skill** governs the workflow (branch → TDD → coder agent → gates → review → PR → CI-green
  merge → close). Delegate code to `coder-opus` (Fable geo-restricted) / `coder-sonnet` for mechanical.
